Objetivo

Desarrollar una solución completa utilizando Azure (y, cuando aplique, Azure OpenAI) que cubra: ingesta de datos, almacenamiento, procesamiento/analítica, exposición mediante API y un componente de IA generativa o de búsqueda semántica.

Resumen

Esta página sintetiza requisitos y criterios de evaluación. Usa la navegación superior para saltar entre secciones.

Puntos clave

  • Arquitectura justificable y trazable a requisitos.
  • Automatización reproducible (IaC / CI/CD cuando aplique).
  • Observabilidad, seguridad y control de costos.

Formato

  • Repositorio con código y documentación.
  • Diagrama(s) y decisiones de diseño.
  • Instrucciones de despliegue/ejecución.

Entregables

  • Documento/README con descripción del problema y alcance.
  • Diagrama de arquitectura (por ejemplo, C4 o similar).
  • Código fuente (APIs, pipelines, notebooks, etc.).
  • Guía de despliegue y ejecución (paso a paso).
  • Explicación de decisiones: servicios, seguridad, costos y trade-offs.
  • Pruebas (unitarias/integración) cuando apliquen.

Rúbrica (criterios de evaluación)

Arquitectura & diseño

  • Selección adecuada de servicios y desacoplo.
  • Seguridad por diseño (identidad, red, secretos).
  • Escalabilidad, resiliencia y observabilidad.

Implementación

  • Calidad de código y estructura del repo.
  • Automatización (IaC/CI/CD) y reproducibilidad.
  • Pruebas, documentación y facilidad de uso.

IA / búsqueda

  • Uso responsable de IA y mitigación de riesgos.
  • Evaluación (métricas/validación) de la solución.
  • Calidad del grounding/citaciones (si aplica).

Operación

  • Costos, límites y gobernanza.
  • Logs, métricas, alertas y trazas.
  • Monitoreo y runbooks básicos.

Recursos

  • Plantillas de arquitectura (C4, Azure Architecture Center).
  • Guías de Well-Architected Framework.
  • Buenas prácticas de seguridad (Zero Trust).
